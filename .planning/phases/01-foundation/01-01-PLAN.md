<task type="auto">
  <name>Task 2: Define complete database schema and seed default roles</name>
  <files>
    src/db/schema.ts
    src/db/index.ts
    src/db/seed.ts
    src/types/index.ts
  </files>
  <action>
1. Create `src/db/index.ts` — Drizzle client singleton:
   - Import `drizzle` from `drizzle-orm/postgres-js` and `postgres` from `postgres`
   - Import `* as schema` from `./schema`
   - Create module-level `client = postgres(process.env.DATABASE_URL!)` — single connection pool
   - Export `db = drizzle(client, { schema })`

2. Create `src/db/schema.ts` with all 6 core tables following the RESEARCH.md code example exactly:
   - `candidateStatusEnum` pgEnum with all 12 statuses: left_to_review, under_review, shortlisted, not_good, maybe, assignment_pending, assignment_sent, assignment_followup, assignment_passed, assignment_failed, hired, rejected
   - `tierEnum` pgEnum with: untiered, junior, senior, both
   - `roles` table: id (uuid pk), name (text), slug (text unique), icon (text default 'Briefcase'), description (text nullable), sortOrder (integer default 0), isActive (boolean default true), createdAt, updatedAt
   - `candidates` table: id (uuid pk), roleId (uuid FK to roles), name (text), email (text nullable), phone (text nullable), instagram (text nullable), portfolioUrl (text nullable), status (candidateStatusEnum default left_to_review), tier (tierEnum default untiered), isDuplicate (boolean default false), importBatchId (uuid FK to importBatches nullable), createdBy (text default 'mock-user'), createdAt, updatedAt
   - `candidateEvents` table: id (uuid pk), candidateId (uuid FK to candidates), eventType (text), fromValue (text nullable), toValue (text), createdBy (text default 'mock-user'), createdAt — INSERT ONLY, no updatedAt
   - `candidateComments` table: id (uuid pk), candidateId (uuid FK to candidates), body (text), createdBy (text default 'mock-user'), createdAt, editedAt (timestamp nullable)
   - `extractionDrafts` table: id (uuid pk), importBatchId (uuid FK to importBatches nullable), sourceUrl (text nullable), rawData (text nullable), extractedData (text nullable — JSON as text), status (text default 'pending'), createdAt
   - `importBatches` table: id (uuid pk), roleId (uuid FK to roles), source (text — 'excel', 'csv', 'paste', 'url', 'manual'), totalRows (integer default 0), importedCount (integer default 0), skippedCount (integer default 0), createdBy (text default 'mock-user'), createdAt

   NOTE: Order tables to avoid forward reference issues: roles → candidates → candidateEvents → candidateComments → importBatches → extractionDrafts. `roles` must be first since `candidates` and `importBatches` reference it. Drizzle uses lazy callback FKs (`.references(() => table.col)`) but the export variable must exist in the module.

3. Create `src/types/index.ts` with inferred types:
   - Export `type Role = typeof roles.$inferSelect`
   - Export `type NewRole = typeof roles.$inferInsert`
   - Export `type Candidate = typeof candidates.$inferSelect`
   - Export `type CandidateEvent = typeof candidateEvents.$inferSelect`
   - Export `type CandidateComment = typeof candidateComments.$inferSelect`
   - Export `type ImportBatch = typeof importBatches.$inferSelect`
   - Export `type ExtractionDraft = typeof extractionDrafts.$inferSelect`
   - Export the enum value arrays as const for reuse: `CANDIDATE_STATUSES`, `TIERS`

4. Create `src/db/seed.ts`:
   - Import `dotenv/config` first
   - Import `drizzle` from `drizzle-orm/postgres-js`, `postgres` from `postgres`
   - Import `roles` from `./schema`
   - Define 4 default roles:
     - Video Editor: slug='video-editor', icon='Film', sortOrder=0
     - Writer/Scriptwriter: slug='writer-scriptwriter', icon='PenLine', sortOrder=1
     - Designer: slug='designer', icon='Palette', sortOrder=2
     - AI/Tech: slug='ai-tech', icon='Cpu', sortOrder=3
   - Use `db.insert(roles).values(defaultRoles).onConflictDoNothing()` to be idempotent (conflict on slug unique constraint)
   - Log result and call `process.exit(0)`

5. Start PostgreSQL: `docker compose up -d`
6. Push schema: `npm run db:push`
7. Seed roles: `npm run db:seed`
  </action>
