  <name>Task 1: Create Server Actions and Zod validation for role CRUD</name>
  <files>
    src/lib/actions/roles.ts
    src/lib/validations/role.ts
  </files>
  <action>
1. Create `src/lib/validations/role.ts`:
   - `roleCreateSchema`: z.object with name (string, min 1, max 100, trimmed), icon (string, min 1), description (string, max 500, optional/nullable).
   - `roleUpdateSchema`: extends roleCreateSchema with id (string uuid), isActive (boolean optional), sortOrder (number int min 0 optional).
   - Export inferred types: `RoleCreateInput`, `RoleUpdateInput`.

2. Create `src/lib/actions/roles.ts` with `'use server'` directive:

   **createRole(formData: FormData):**
   - Parse with roleCreateSchema.safeParse. Return `{ error: fieldErrors }` on failure.
   - Generate slug: `name.toLowerCase().replace(/[^a-z0-9]+/g, '-').replace(/^-|-$/g, '')`. This handles special characters like "/" in "AI/Tech".
   - Check slug uniqueness: query roles table for existing slug. If exists, return `{ error: { name: ['A role with this name already exists'] } }`.
   - Get max sortOrder from roles table, add 1 for new role's sortOrder.
   - Insert with `db.insert(roles).values({ name, slug, icon, description, sortOrder })`.
   - Call `revalidatePath('/', 'layout')` to invalidate sidebar cache globally.
   - Return `{ success: true }`.

   **updateRole(formData: FormData):**
   - Parse with roleUpdateSchema.safeParse (id comes from hidden form field).
   - If name changed, regenerate slug and check uniqueness (excluding current role's id).
   - Update with `db.update(roles).set({ name, slug, icon, description, updatedAt: new Date() }).where(eq(roles.id, id))`.
   - Call `revalidatePath('/', 'layout')`.
   - Return `{ success: true }`.

   **toggleRoleActive(roleId: string):**
   - Fetch current role by id.
   - If currently active and about to deactivate: check if role has candidates (`db.select({ count: count() }).from(candidates).where(eq(candidates.roleId, roleId))`). If count > 0, return `{ error: 'Cannot deactivate a role that has candidates. Reassign or remove candidates first.' }`.
   - Toggle isActive, set updatedAt.
   - Call `revalidatePath('/', 'layout')`.
   - Return `{ success: true }`.

   **reorderRoles(roleIds: string[]):**
   - Accept an ordered array of role IDs.
   - For each id at index i, update sortOrder to i.
   - Call `revalidatePath('/', 'layout')`.
   - Return `{ success: true }`.

   All actions use immutable patterns — create new objects, never mutate existing ones. All actions handle errors explicitly and return structured results (never throw unhandled).
  </action>
