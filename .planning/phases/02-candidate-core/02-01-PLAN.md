**src/lib/queries/candidates.ts — getCandidates function:**

Interface GetCandidatesParams:
- roleId: string (required, omit for master view by making optional)
- page: number (default 1)
- status: CandidateStatus[] (default [])
- tier: Tier | null (default null)
- sort: 'newest' | 'oldest' | 'name_asc' | 'updated' (default 'newest')
- q: string (default '')
- dateRange: 'today' | 'week' | 'month' | null (default null)
- duplicatesOnly: boolean (default false)

Implementation:
- Build conditions array starting with roleId filter (if provided)
- Conditionally add: inArray for status, eq for tier, ilike OR on name+email+phone for search (SRCH-01 requires name, email, AND WhatsApp/phone), gte/lte for date ranges (compute from current date), eq isDuplicate for duplicatesOnly
- CRITICAL: The search `q` filter must use `or(ilike(candidates.name, ...), ilike(candidates.email, ...), ilike(candidates.phone, ...))` — all three fields, not just name+email. SRCH-01 explicitly requires WhatsApp/phone search.
- CRITICAL: Guard all conditions — never pass undefined to Drizzle where clauses (see Research pitfall 3)
- Compute dateRange boundaries: 'today' = start of today, 'week' = 7 days ago, 'month' = 30 days ago
- orderBy mapping: newest=desc(createdAt), oldest=asc(createdAt), name_asc=asc(name), updated=desc(updatedAt)
- Run count query with SAME conditions (shared array — see Research pitfall 6)
- Run data query with same conditions + orderBy + limit(50) + offset((page-1)*50)
- Return { candidates: data, total, page, totalPages: Math.ceil(total/50) }

Also create a getCandidateWithEvents function:
- Takes candidateId: string
- Fetches candidate by id
- Fetches all candidateEvents for that candidate ordered by createdAt desc
- Returns { candidate, events }
- NOTE: This function uses Drizzle directly and CANNOT be called from client components. The fetchCandidateProfile server action in candidates.ts wraps this for client-side use.

**src/hooks/use-debounce.ts:**

- Generic type parameter, accepts value and delay (ms)
- Returns debounced value

Do NOT install use-debounce package — hand-roll per the skill/research recommendation.
