**candidate-filter-bar.tsx — CandidateFilterBar component:**
- Props: none — reads all state from URL search params via useSearchParams
- Uses useRouter and usePathname to update URL params on filter change
- Helper function setFilter(key, value): creates new URLSearchParams, sets/deletes key, always deletes 'page' on filter change (reset to page 1), calls router.push

Layout: horizontal bar, flex, items-center, gap-3, between the tab strip and table. Always visible per CONTEXT.md.

Components (left to right):
1. **Status multi-select dropdown:**
   - Use DropdownMenu from UI kit
   - Trigger button shows "Status" + count badge if any selected (e.g., "Status (3)")
   - Content: checkbox items for each of the 12 statuses, using DropdownMenuCheckboxItem or similar
   - Selected statuses stored as comma-separated in URL param ?status=hired,rejected
   - Clicking a status toggles it in the comma list

2. **Tier pill buttons:**
   - Row of pill/toggle buttons: All, Untiered, Junior, Senior, Both
   - "All" means no tier filter (delete param)
   - Active pill gets highlighted styling (bg-blue-100 text-blue-700)
   - Sets ?tier=junior etc in URL

3. **Date Added dropdown:**
   - Button trigger: "Date" + count badge if active
   - Options: Today, This Week (7d), This Month (30d), All Time (clear)
   - Sets ?date=today|week|month in URL

4. **Sort dropdown:**
   - Button trigger: "Sort: [current]"
   - Options: Newest (default), Oldest, Name A-Z, Last Updated
   - Sets ?sort=newest|oldest|name_asc|updated in URL

5. **Show Duplicates toggle:**
   - Simple checkbox or toggle: "Duplicates only"
   - Sets ?duplicates=true in URL

6. **Search input (right-aligned):**
   - Text input with search icon, placeholder "Search name or email..."
   - Uses useDebounce hook (300ms) from @/hooks/use-debounce
   - On debounced value change, sets ?q=searchterm in URL
   - Controlled input — reads initial value from searchParams.get('q')

7. **Active filter summary (below or inline):**
   - Count of active filters shown as badge
   - "Clear all" / "Reset" button appears when any filter is active
   - Clicking clears all filter params (push base pathname only)
   - Shows "Showing X of Y candidates" text — props: showing (number), total (number)

Add showing/total as props to the filter bar OR render this text in the parent. Simplest: add these as props.
