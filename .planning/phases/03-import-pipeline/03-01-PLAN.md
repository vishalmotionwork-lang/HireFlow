Create `src/lib/import/types.ts` with these types:

- `RawRow`: `unknown[]` (a single row as array of cell values)
- `ParseResult`: `{ headers: string[]; rows: RawRow[] }` (output of any parser)
- `CandidateField`: union type `'name' | 'email' | 'phone' | 'instagram' | 'portfolioUrl' | 'ignore'`
- `ColumnMapping`: `Partial<Record<Exclude<CandidateField, 'ignore'>, number>>` (maps field to column index)
- `NormalizedRow`: `{ name: string | null; email: string | null; phone: string | null; instagram: string | null; portfolioUrl: string | null; _rowIndex: number }`
- `RowError`: `{ field: string; message: string }`
- `ValidatedRow`: `NormalizedRow & { errors: RowError[]; isValid: boolean }`
- `DuplicateMatch`: `{ id: string; name: string; roleId: string }`
- `DuplicateInfo`: `{ matchType: 'email' | 'phone'; existing: DuplicateMatch }`
- `ImportRowDecision`: `'import' | 'merge' | 'skip'`
- `ImportResult`: `{ batchId: string; importedCount: number; skippedCount: number; mergedCount: number; duplicatesFound: number; totalRows: number }`

<task type="auto">
  <name>Task 2: Build parsing, heuristic, normalization, and validation utilities</name>
  <files>src/lib/import/parseExcel.ts, src/lib/import/parseCsv.ts, src/lib/import/columnHeuristics.ts, src/lib/import/normalizeRows.ts, src/lib/import/validateRows.ts</files>
  <action>
Create 5 utility files. All are pure functions, no side effects, no DB access.

**parseExcel.ts** — Client-only ('use client' is NOT needed since it's a lib, but it must only be imported from client components). Export `parseExcelFile(file: File): Promise<ParseResult>`. Use `XLSX.read(arrayBuffer)` with `sheet_to_json(worksheet, { header: 1 })`. First row = headers (map to strings via `String(h ?? '').trim()`), rest = data rows. Filter out completely empty rows (all cells undefined/empty string). See RESEARCH.md Pattern 1 for exact code.

**parseCsv.ts** — Export two functions:
1. `parseCsvString(input: string): ParseResult` — for paste import. Strip BOM `\ufeff` before parsing. Use `Papa.parse(cleaned, { header: false, skipEmptyLines: true, delimiter: '' })`. First row = headers, rest = data.
2. `parseCsvFile(file: File): Promise<ParseResult>` — for file upload. Same options but uses PapaParse's file mode with `complete` callback wrapped in Promise.

**columnHeuristics.ts** — Export `detectMapping(headers: string[]): ColumnMapping`. Keyword matching against FIELD_KEYWORDS map:
- name: ['name', 'full name', 'candidate', 'applicant', 'person']
- email: ['email', 'e-mail', 'mail']
- phone: ['phone', 'mobile', 'whatsapp', 'contact', 'number', 'ph', 'wa']
- instagram: ['instagram', 'ig', 'insta', 'handle']
- portfolioUrl: ['portfolio', 'link', 'url', 'website', 'work', 'behance', 'dribbble', 'youtube']

Normalize header to lowercase, check `includes(keyword)`. First match wins per field (don't double-assign). Return partial mapping — unmapped fields stay undefined.

**normalizeRows.ts** — Export `normalizeRows(rows: RawRow[], mapping: ColumnMapping): NormalizedRow[]`. Map each row using column indices from mapping. Include `normalizePhone` helper (strip non-digits, handle +91/91 prefix → last 10 digits for Indian numbers, keep 7+ digit international). Normalize emails to lowercase. Trim all string values. Empty strings → null.

**validateRows.ts** — Export `validateRows(rows: NormalizedRow[]): ValidatedRow[]`. Use Zod schema: name is `z.string().min(1, 'Name is required')`, email is `z.string().email().nullable().optional()`, portfolioUrl is `z.string().url().nullable().optional()`, phone and instagram are `z.string().nullable().optional()`. Return each row with `errors` array and `isValid` boolean.

IMPORTANT: Name is required (IMPT-07 flags missing name). Portfolio URL is NOT required but should be valid URL if present.
