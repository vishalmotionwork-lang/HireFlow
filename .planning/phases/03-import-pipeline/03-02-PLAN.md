**Types for the action input** (define locally or import from types if available):
```typescript
interface ImportRow {
  name: string;
  email: string | null;
  phone: string | null;
  instagram: string | null;
  portfolioUrl: string | null;
  decision: 'import' | 'merge' | 'skip';
  mergeTargetId?: string; // candidate ID to merge into (when decision='merge')
}
```

**Export `importCandidates` server action:**
```typescript
export async function importCandidates(
  rows: ImportRow[],
  targetRoleId: string,
  source: 'excel' | 'csv' | 'paste'
): Promise<ImportResult>
```

Implementation steps:
1. Validate `targetRoleId` is a valid UUID and exists in roles table
2. Create `importBatches` record with `roleId`, `source`, `totalRows: rows.length`, `importedCount: 0`, `skippedCount: 0`
3. Separate rows by decision: toInsert (decision='import'), toMerge (decision='merge'), toSkip (decision='skip')
4. For `toInsert` rows: bulk insert into candidates with `importBatchId: batch.id`, `isDuplicate: false`, status defaults to 'left_to_review'
5. For `toMerge` rows: for each, update the existing candidate (mergeTargetId) — set non-null incoming fields where existing field is null (prefer existing data, fill gaps). Set `isDuplicate: true` on the existing record. Use a transaction for atomicity.
6. For each inserted/merged candidate, insert a 'created' or 'imported' event into candidateEvents
7. Update the importBatches record: `importedCount = toInsert.length + toMerge.length`, `skippedCount = toSkip.length`
8. Call `revalidatePath('/', 'layout')`
9. Return `ImportResult` object with batchId, counts

**Export `detectDuplicates` function** (called by wizard UI before final import):
```typescript
export async function detectDuplicates(
  emails: string[],
  phones: string[]
): Promise<Record<string, DuplicateMatch>>
```

Use single Drizzle query with `or(inArray(candidates.email, emails), inArray(candidates.phone, phones))`. Build lookup map keyed by `email:value` and `phone:value`. Join with roles table to get role name for the duplicate message ("may already exist as [Name] in [Role]"). Return the map.

**Error handling:** Wrap in try/catch, return `{ error: string }` on failure. Use MOCK_USER.name for createdBy (consistent with existing actions).

**IMPORTANT:** Do NOT auto-merge. The `decision` field on each row is set by the user in the wizard Step 3 UI. This action just executes their decisions.
