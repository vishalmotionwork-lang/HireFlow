**ImportWizard.tsx** — Client component (`'use client'`). Props: `{ roles: Role[] }`.

Implement with `useReducer`:
- State type `WizardState`: `{ step: 'upload' | 'map' | 'validate' | 'summary'; rawRows: RawRow[]; headers: string[]; mapping: ColumnMapping; targetRoleId: string; validatedRows: ValidatedRow[]; duplicateInfo: Record<number, DuplicateInfo>; result: ImportResult | null }`
- Actions: `FILE_PARSED`, `MAPPING_CONFIRMED`, `VALIDATION_COMPLETE`, `IMPORT_COMPLETE`, `BACK`, `RESET`
- Render step indicator (4 dots/labels: Upload, Map, Validate, Summary) showing current step highlighted
- Conditionally render Step1Upload, Step2Mapping, Step3Validate, Step4Summary based on `state.step`
- Step3Validate and Step4Summary will be created in Plan 04 — for now, render placeholder `<div>Step 3/4 coming soon</div>` so the wizard compiles

Store wizard state in `sessionStorage` keyed by `'hireflow-import-wizard'` — restore on mount, clear on RESET or successful import. This prevents data loss on accidental navigation (Pitfall 6 from research).

**Step1Upload.tsx** — Client component. Props: `{ onParsed: (result: ParseResult) => void }`.

Two tabs (use base-ui Tabs or simple button toggle):
1. **File Upload tab**: Use `react-dropzone` for drag-and-drop zone. Accept `.xlsx, .xls, .csv`. Show accepted file types label. On drop/select:
   - If .xlsx or .xls: call `parseExcelFile(file)` from `src/lib/import/parseExcel.ts`
   - If .csv: call `parseCsvFile(file)` from `src/lib/import/parseCsv.ts`
   - On success: call `onParsed(result)`
   - On error: show error toast via sonner
   - Show file name + row count after successful parse
2. **Paste tab**: Large textarea (min 6 rows height). "Paste" button below. On click: call `parseCsvString(textareaValue)` from `src/lib/import/parseCsv.ts`, then `onParsed(result)`.

Style the dropzone: dashed border, centered icon (Upload from lucide-react), "Drag & drop or click to browse" text. Active/hover state with blue border.

**Step2Mapping.tsx** — Client component. Props: `{ headers: string[]; rows: RawRow[]; roles: Role[]; onConfirm: (mapping: ColumnMapping, targetRoleId: string) => void; onBack: () => void }`.

Layout:
1. **Role selector** at top: dropdown populated with `roles`. User selects target role for all imported candidates. If headers contain a column that matches role keywords ('role', 'position', 'applied for'), show a note that role assignment from sheet is not yet supported — they must pick one role. Preselect first role.
2. **Preview table**: Show first 5 rows of data in a table. Columns match the raw headers.
3. **Mapping row**: Above (or below) the preview, show a row of dropdowns — one per header column. Each dropdown has options: Name, Email, Phone, Instagram, Portfolio URL, Ignore. Auto-populate using `detectMapping(headers)` from columnHeuristics. User can change any mapping.
4. **Unmapped required fields warning**: If `name` is not mapped to any column, show amber warning: "Name column is required"
5. **Confirm button**: disabled if name not mapped. On click: call `onConfirm(mapping, selectedRoleId)`

Style with existing Tailwind patterns — card container, table with borders, dropdowns matching shadcn select style.
