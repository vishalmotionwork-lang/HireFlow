  <files>src/components/import/Step3Validate.tsx, src/components/import/Step4Summary.tsx</files>
  <action>
**Step3Validate.tsx** — Client component. Props: `{ rows: RawRow[]; headers: string[]; mapping: ColumnMapping; targetRoleId: string; roles: Role[]; source: 'excel' | 'csv' | 'paste'; onImportComplete: (result: ImportResult) => void; onBack: () => void }`.

On mount (useEffect):
1. Call `normalizeRows(rows, mapping)` to get NormalizedRow[]
2. Call `validateRows(normalizedRows)` to get ValidatedRow[]
3. Collect all non-null emails and phones from rows
4. Call `detectDuplicates(emails, phones)` server action to get duplicate map
5. Cross-reference: for each row, check if its email or phone appears in duplicate map. If yes, attach DuplicateInfo.

Render a scrollable table showing all rows with columns: Row #, Name, Email, Phone, Instagram, Portfolio URL, Status, Action.

**Status column logic:**
- Valid row, no duplicate: green checkmark "Ready"
- Valid row, has duplicate: yellow warning icon + "Duplicate — may already exist as [Name] in [Role]" (DUPL-02)
- Invalid row (errors): red X icon + first error message (e.g., "Name is required")

**Action column logic:**
- Valid, no duplicate: checkbox (checked by default) to include/exclude
- Valid, has duplicate: dropdown with three options: "Import as new" (default), "Merge with [Name]", "Skip"
- Invalid: "Skip" label (greyed out, cannot import invalid rows). Or allow user to manually fix the row value inline if they want — but simpler to just skip and let them add manually after.

**Summary bar at bottom:** "X ready to import, Y duplicates found, Z will be skipped (invalid)"

**Import button:** "Import X Candidates" — disabled while importing. On click:
1. Build ImportRow[] array from user decisions
2. Call `importCandidates(importRows, targetRoleId, source)` server action
3. On success: call `onImportComplete(result)`
4. On error: show error toast via sonner

**Loading state:** Show spinner/progress during import. Disable all controls.

**Step4Summary.tsx** — Client component. Props: `{ result: ImportResult; onStartNew: () => void; targetRoleId: string; roles: Role[] }`.

Render a summary card with:
- Green checkmark header: "Import Complete"
- Stats grid: Imported (green number), Skipped (grey), Merged (blue), Duplicates Found (amber)
- Total rows processed
- Two action buttons:
  1. "View Candidates" — links to `/roles/[slug]` for the target role (look up slug from roles array)
  2. "Import More" — calls onStartNew (resets wizard)
