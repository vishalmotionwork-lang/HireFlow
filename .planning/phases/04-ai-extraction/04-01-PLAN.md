<task type="auto">
  <name>Task 2: Upgrade extraction actions to async after() + add polling endpoint</name>
  <files>
    src/lib/actions/extraction.ts
    src/app/api/extraction-status/[batchId]/route.ts
  </files>
  <action>
1. Upgrade `src/lib/actions/extraction.ts`:
   - Add new `startExtractions(urls: string[], roleId: string)` server action:
     a. Validate: urls array is non-empty, max 20 URLs per batch, roleId exists in DB
     b. Create importBatch row with `source: 'url'`, `totalRows: urls.length`
     c. Create extractionDraft rows for each URL with `status: 'pending'`, `importBatchId: batch.id`
     d. Use `after()` from `next/server` for fire-and-forget background processing
     e. Inside `after()`: iterate drafts sequentially, for each:
        - Update status to 'processing'
        - Call `scrapeUrl(draft.sourceUrl)` from firecrawl.ts
        - If scrape fails: update status='failed', error=reason, continue to next
        - If scrape succeeds: call `runExtraction({ rawText: scraped.markdown, sourceUrl: draft.sourceUrl })`
        - Update draft: status='completed', extractedData=result.data, fieldConfidence=result.fieldConfidence, overallConfidence=Math.round(result.overallConfidence * 100), platform=result.platform
        - Catch per-URL errors individually — one URL failure must not stop the batch
     f. Return `{ batchId: batch.id }` immediately (before after() runs)

   - Add `startSingleExtraction(url: string, roleId: string)` for IMPT-09:
     a. Same as startExtractions but with a single URL
     b. Creates candidate with name='Pending extraction...', roleId, source='url', portfolioUrl=url
     c. Creates extractionDraft with both candidateId and importBatchId
     d. Uses after() same way
     e. Returns `{ batchId, candidateId }`

   - Add `confirmExtraction(draftId: string, edits: Record<string, string>)`:
     a. Load draft, verify status is 'completed'
     b. If draft has candidateId: update candidate with extracted+edited fields (use applyExtraction pattern already in the file but merge with edits)
     c. If draft has no candidateId (bulk flow): create new candidate from extracted data + edits
     d. Set draft status='applied', appliedAt=new Date()
     e. Log candidateEvent for the creation/update

   - Add `skipExtraction(draftId: string)`:
     a. Set draft status='reviewed', reviewedAt=new Date()
     b. No candidate changes

   - Keep existing functions (createExtractionDraft, processExtraction, etc.) for backward compat but the new flow uses startExtractions.

2. Create `src/app/api/extraction-status/[batchId]/route.ts`:
   - GET handler: await params, extract batchId
   - Query extractionDrafts where importBatchId = batchId
   - Return JSON: `{ total, done, pending, failed, drafts: [{ id, sourceUrl, status, extractedData, error, overallConfidence }] }`
   - `done` = count of status in ('completed', 'failed', 'applied', 'reviewed')
   - `pending` = count of status in ('pending', 'processing')
   - `failed` = count of status === 'failed'
  </action>

    - Bulk paste capped at 20 URLs
