  <name>Task 1: Build ExtractionReviewModal + ExtractionReviewList</name>
  <files>
    src/components/import/ExtractionReviewModal.tsx
    src/components/import/ExtractionReviewList.tsx
  </files>
  <action>
1. Create `src/components/import/ExtractionReviewList.tsx` (client component):
   - Props: `{ drafts: ExtractionDraft[]; onSelectDraft: (draftId: string) => void; selectedDraftId: string | null }`
   - Render a scrollable list of all drafts in the batch
   - Each row shows:
     a. Source URL (truncated, clickable to open in new tab)
     b. Status badge: "Ready" (completed), "Failed" (failed), "Applied" (applied), "Skipped" (reviewed)
     c. Overall confidence as a colored badge using `getConfidenceLabel()` + `getConfidenceColor()` from confidence.ts
   - Completed (status='completed') drafts are clickable — calls `onSelectDraft(draftId)`
   - Failed drafts show error message inline, not clickable
   - Applied/reviewed drafts show as dimmed/done state
   - Summary at top: "X of Y reviewed" progress

2. Create `src/components/import/ExtractionReviewModal.tsx` (client component):
   - Props: `{ draft: ExtractionDraft; onConfirm: () => void; onSkip: () => void; onNext: () => void; onPrev: () => void; hasNext: boolean; hasPrev: boolean }`
   - Parse `draft.extractedData` (jsonb) and `draft.fieldConfidence` (jsonb) from the draft
   - Render a card/modal layout with the source URL at the top (clickable, opens new tab)
   - For each field (name, email, phone, instagram, bio, location):
     a. Label on the left
     b. Editable text input with the extracted value (or empty string if null)
     c. Confidence badge on the right: use `getConfidenceLabel(score)` and `getConfidenceColor(score)` — green/yellow/orange/red
     d. If value is null/empty: show input with red border and "Not found" placeholder text
   - For portfolioLinks and socialHandles: render as a list of chips/tags (non-editable in v1 — extracted links shown as-is)
   - Overall confidence indicator at top of card
   - Action buttons at bottom:
     a. "Skip" (muted/secondary) — calls `skipExtraction(draftId)` then `onSkip()`
     b. "Confirm and Save" (primary) — collects all edited field values, calls `confirmExtraction(draftId, editedFields)` then `onConfirm()`
     c. Navigation: "< Prev" and "Next >" buttons for batch review (only shown if hasPrev/hasNext)
   - Loading state on confirm/skip buttons while server action runs
   - Use `useActionState` or `useTransition` for pending states (follow existing project patterns)
   - Error handling: sonner toast on server action failure

3. **Field editing approach:**
   - Use local React state for each field (initialized from extractedData on mount)
   - On Confirm: collect all local state values into an edits object, pass to confirmExtraction
   - This is immutable — the draft.extractedData is never mutated, only the local state changes
  </action>

    - All fields are editable via text inputs
    - Confirm calls confirmExtraction with edited values
    - Skip calls skipExtraction
    - Navigation between drafts in batch via Prev/Next
    - Loading states on action buttons
  </done>
